# TECNOLOGÍAS
- Backend: Node.js/Express + SQL Server + .NET 7 API (EF Core)
- Frontend: HTML5/CSS3/JS vanilla (sin frameworks)
- Seguridad: bcrypt hashing contraseñas, Stripe pagos
- Base datos: SQL Server con transacciones ACID
- Responsive: CSS Grid/Flexbox, Mobile-first

## 4. db/script_base_datos.sql (Script BD Completo)
```
-- Crear BD OrdenExpress
CREATE TABLE Cliente(ID_Cliente INT PK IDENTITY, Nombre, Apellido_Paterno, Usuario, Contraseña, Foto_Perfil);
CREATE TABLE Administrador(...);
CREATE TABLE Producto(ID_Producto PK, Nombre_P, Precio_Venta, Cantidad_Disponible, Imagen);
CREATE TABLE Pedido(ID_Pedido PK, ID_Cliente FK, Estado, Total, Metodo_Pago);
CREATE TABLE Detalle_Pedido(ID_Detalle PK, ID_Pedido FK, ID_Producto FK, Cantidad);
INSERT Admin default (admin/admin);
```

## 7. Frontend HTML (6 páginas completas)
- **index.html**: Carrusel productos, preview menú, llamadas API /cliente/carrusel /producto/menu
- **login.html**: Form login → AuthService.login()
- **registro.html**: Form registro → POST /cliente con validaciones
- **menu.html**: Lista productos con modal detalle → agregar al carrito
- **carrito.html**: Gestión carrito, Stripe/efectivo pagos → PedidoService.crear()
- **cocina.html**: Panel cocina real-time, filtros estados, PUT /pedido/estado/:id

## 8. Routes Node.js (5 archivos ~1000 líneas)
- **cliente.js**: POST registro/login (bcrypt), PUT fotoPerfil, GET carrusel, GET /:id
- **producto.js**: POST upload imagen, GET lista/menú/inventario, PUT entrada/:id, PUT /:id
- **pedido.js**: POST crear (transacción SQL), PUT estado/:id, GET /cliente/:id, GET todos
- **pagos.js**: POST stripe sesión/webhook, POST pago-efectivo, GET metodos
- **reportes.js**: GET ventas diarias/semanales/mensuales, top productos, resumen estadísticas
